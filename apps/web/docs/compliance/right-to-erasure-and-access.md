# Right to Erasure and Access — Guide

Last updated: 2025-09-04

## Purpose
Explain how to handle DSAR and erasure requests in Nexa.

## Who should read this
Support, privacy, and engineering teams.

## Steps
1) Intake the request. Confirm identity.
2) Verify authority. Log the request ID.
3) Scope data sets. Minimise collection.
4) Fulfil DSAR: export and redact where needed (`***redacted***`).
5) Erasure: delete or pseudonymise, respecting legal holds.
6) Record an audit log entry.
